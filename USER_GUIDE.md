# 📘 부산대학교 AI융합교육원 - 사용자 가이드 (User Guide)

이 문서는 **2026-ai-guidebook (부산대학교 AI융합교육원 온라인 브로슈어)** 사이트의 완벽한 활용 및 유지보수 방법을 안내하는 공식 사용자 가이드입니다.

본 사이트는 Docusaurus 기반으로 제작되었으며, **독자적인 웹 기반 관리자 에디터(CMS)**가 탑재되어 있어 코딩 지식이 없어도 누구나 웹에서 직접 브로슈어를 수정하고 실시간으로 배포할 수 있습니다.

---

## 1. 관리자 에디터 사용법 (문서 수정 및 배포)

기존 브로슈어 내용을 수정하거나, 새로운 파트를 추가하고 싶을 때는 로컬 편집기(VSCode 등) 없이 웹에서 바로 처리할 수 있습니다.

### 1.1. 관리자 페이지 접속하기
두 가지 방법으로 관리자 에디터에 접속할 수 있습니다:
1. **버튼을 통한 접속**: 홈페이지 각 브로슈어 문서의 맨 아래쪽 스크롤을 내리면 나타나는 **[이 페이지 편집하기]** 파란색 프리미엄 버튼을 클릭합니다. (보고 있던 문서가 자동으로 불러와집니다)
2. **직접 접속**: 주소창 끝에 `/admin/index.html`을 붙여서 접속합니다.
   - 예시: `https://pnu-ai-center.github.io/2026-ai-guidebook/admin/index.html`

### 1.2. ⚙️ GitHub 토큰 (PAT) 발급 및 설정하기 (최초 1회 필수)
문서를 저장하고 배포하려면 GitHub의 권한이 필요합니다. 
1. 관리자 페이지 우측 상단의 **[⚙️ 톱니바퀴 아이콘]**을 클릭합니다.
2. `[토큰 발급받으러 가기]` 링크를 눌러 GitHub에서 새로운 Token(Classic)을 생성합니다.
   - 권한(Scopes) 설정 시 반드시 **`repo`** 항목에 체크해야 합니다.
3. 발급받은 `ghp_...` 로 시작하는 토큰을 빈칸에 붙여넣고 [저장]을 누릅니다.
   - *이 토큰은 현재 사용 중인 브라우저에 안전하게 저장됩니다.*

### 1.3. 실시간 미리보기(Live Preview)와 문서 수정 🎨
- 브로슈어를 수정하기 위해 문서를 열면 **[Markdown]** 탭이 기본으로 선택되어 열립니다.
- 브로슈어 파일들은 매우 복잡한 레이아웃(그리드, 색상, 애니메이션 등)을 포함하고 있습니다. **절대 `WYSIWYG(서식 있는 텍스트)` 탭을 누르지 마시고, `Markdown` 탭에서 글자 내용만 수정**하세요.
- **실시간 미리보기**: `Markdown` 탭에서 글씨를 수정하면, 우측(또는 탭 전환 시) 미리보기 화면에 **실제 브로슈어와 똑같은 100% 동일한 디자인과 색상**으로 변경 사항이 실시간 렌더링됩니다! 마음 놓고 수정 후 확인하세요.

### 1.4. 저장 및 자동 배포 🚀
1. 수정을 마쳤다면 우측 하단의 **[저장 및 배포]** 버튼을 클릭합니다.
2. 완료 메시지와 함께 깃허브 액션(GitHub Actions)이 자동으로 백그라운드에서 빌드를 시작합니다.
3. 약 **1~2분 뒤**, 실제 홈페이지(GitHub Pages)에 수정된 내용이 완벽하게 반영됩니다.

## 2. 브로슈어 컴포넌트(디자인 블록) 활용 가이드

새로운 페이지를 만들거나 기존 페이지에 예쁜 디자인 요소를 추가하고 싶을 때 사용할 수 있는 **기본 템플릿 코드**들입니다.

> 💡 관리자 에디터에 아래 코드를 그대로 복사(Ctrl+C)해서 붙여넣기(Ctrl+V) 한 뒤, 텍스트만 고쳐서 사용하세요! 
> (**홈페이지의 `템플릿 및 컴포넌트 가이드` 메뉴에서도 확인하실 수 있습니다.**)

### 2.1. 하이라이트 글씨 (Highlight)
원하는 색상으로 글자를 강조하고 싶을 때 사용합니다.
```jsx
- <Highlight color="#1651c8">파란색 하이라이트</Highlight>
- <Highlight color="#e84b35">빨간색 하이라이트</Highlight>
```

### 2.2. 반응형 다단 레이아웃 (Columns)
PC에서는 가로로 나란히, 스마트폰에서는 세로로 접히는 반응형 레이아웃입니다.
```jsx
<Columns>
  <Column>
    <h3>✨ 첫 번째 컬럼</h3>
    <p>이곳에 내용을 적어주세요.</p>
  </Column>
  <Column>
    <h3>🚀 두 번째 컬럼</h3>
    <p>컬럼을 하나 더 추가하면 3단 레이아웃이 됩니다.</p>
  </Column>
</Columns>
```

### 2.3. 단계별 로드맵 및 구역 나누기 (Phase Blocks)
사업의 1단계, 2단계처럼 시간 순서나 구역을 세련된 회색/흰색 박스로 나눌 때 사용합니다. 스크롤을 내릴 때 스르륵 올라오는 애니메이션(`fade-up`)이 기본 적용됩니다.
```html
<div className="phase-block fade-up">
  <div className="phase-block-header">
    <h3>1단계: 타이틀 적기</h3>
    <span className="period-tag">2023–2024</span>
  </div>
  <div className="phase-block-body">
    <div className="area-group">
      <div className="area-label">교육 혁신</div>
      <ul className="item-list">
        <li>전공트랙 신설 및 교육과정 개편</li>
        <li>내용을 추가하세요</li>
      </ul>
    </div>
  </div>
</div>
```

## 3. 사이트 구조 및 핵심 기능 설명

이 사이트는 단순한 텍스트 기반 위키가 아닌, **마케팅/홍보용 고퀄리티 브로슈어** 역할을 하도록 특수하게 커스터마이징 되었습니다.

### 3.1. 자동 생성 목차와 스크롤 트래킹 (TOC Scroll Spy)
- **우측 플로팅 목차**: 사용자가 문서를 읽으며 스크롤을 내릴 때, 오른쪽 목차 메뉴가 화면에 고정되어 따라다닙니다.
- 브로슈어 내부의 화면 위치를 정확하게 인식하여, 현재 읽고 있는 파트의 목차 항목이 **진한 파란색으로 하이라이트** 처리됩니다.
- 이는 내부적으로 보이지 않는 투명한 마크다운 헤딩(`##`)을 각 구역마다 자동으로 삽입하여 구현한 고도의 커스텀 트릭입니다.

### 3.2. 프리미엄 네비게이션 바
- 화면 상단의 드롭다운 메뉴는 사용자가 마우스를 올렸을 때 부드러운 전환 효과(Fade In)와 그림자를 동반하며 열립니다.
- 마우스가 틈새(Gap)를 지날 때 메뉴가 닫히는 고질적인 버그를 완벽하게 차단하도록 설계되었습니다.

### 3.3. 정보 밀도 극대화 (Information Density)
- 여러 번 스크롤할 필요 없이 한눈에 많은 정보를 볼 수 있도록, 폰트 크기, 문단 여백, 줄 간격이 콤팩트하고 세밀하게 튜닝되어 있습니다.

## 4. 자주 묻는 질문 (FAQ)

**Q. 글자를 고치고 [저장 및 배포]를 눌렀는데 사이트가 안 변해요!**  
A. GitHub Actions에서 서버에 새 문서를 만들어 올리는 데 보통 **1분에서 2분** 정도 소요됩니다. 여유를 가지고 기다리신 후 브라우저 새로고침(F5)을 해보세요.

**Q. 로고 이미지를 바꾸고 싶어요.**  
A. 깃허브의 `static/img/` 폴더에 원하는 이미지를 업로드(예: `logo.jpg`)하신 뒤, `docusaurus.config.js` 파일에서 `logo.src` 항목의 파일명을 교체해 주시면 됩니다.

**Q. 관리자 에디터에서 WYSIWYG 탭을 실수로 눌렀어요! 어떻게 하죠?**  
A. 저장 버튼을 누르지 마시고 페이지를 그냥 새로고침 하시거나 닫아버리세요. 변경사항이 저장되지 않아 안전합니다.

## 1. 기초 이해: PNU Slate의 전체 구조와 작동 원리

PNU Slate는 단순한 웹사이트가 아니라, 여러 교육 프로그램을 체계적으로 관리하기 위한 **세 가지 핵심 레이어**로 구성되어 있습니다.

### 1.1 서비스 구성도 (어떤 페이지들이 있나요?)

* **PNU AI Hub (허브 페이지)**: 모든 브로슈어 프로젝트가 모여 있는 **관리 센터**입니다. 여기에서 원하는 교육 프로그램(레포지토리)을 선택하여 진입할 수 있습니다.
* **Admin Page (관리자 페이지)**: 실제 내용을 수정하고 저장하는 **작업실**입니다. 프로그래밍 지식이 없어도 마우스와 텍스트 입력만으로 브로슈어를 편집할 수 있습니다.
* **Brochure Page (브로슈어 페이지)**: 편집이 완료되어 세상에 공개된 **최종 결과물**입니다. 학생들이 실제로 접속하여 정보를 확인하는 사이트입니다.

### 1.2 시스템 작동 원리 (Git 기반 CMS)

PNU Slate는 일반적인 홈페이지와 달리 'Git 기반 CMS'라는 방식으로 작동합니다. 편집자가 수정 버튼을 누른 후 실제 사이트에 반영되기까지의 과정은 다음과 같습니다.

1. **선택 및 진입**: **허브 페이지**에서 수정할 프로젝트를 선택해 **관리자 페이지**로 들어갑니다.
2. **저장 및 기록**: 관리자 페이지에서 내용을 고치고 [저장]을 누르면, 그 즉시 GitHub(코드 저장소)라는 안전한 금고에 수정 기록이 남습니다.
3. **자동 조립**: 금고에 새 기록이 들어오면, **GitHub Actions**라는 로봇이 약 1~2분 동안 홈페이지를 다시 조립(빌드)합니다.
4. **최종 공개**: 조립이 끝나면 **[pnuai.github.io](https://pnuai.github.io)** 주소를 통해 전 세계에 공개됩니다.

## 2. 준비 단계: 관리 권한(GitHub 토큰) 확보하기

가장 먼저 해야 할 일은 "나는 관리자가 맞다"라는 열쇠(토큰)를 만드는 것입니다. 딱 한 번만 수행하면 됩니다.

### 2.1 개인 액세스 토큰(PAT) 발급

1. [GitHub](https://github.com/)에 로그인합니다.
2. 우측 상단 프로필 클릭 -> **Settings** -> 좌측 맨 아래 **Developer settings** 클릭.
3. **Personal access tokens** -> **Tokens (classic)** 선택.
4. **Generate new token (classic)** 버튼을 누릅니다.
5. **Note**: `PNU Slate 관리용`이라고 적습니다.
6. **Select scopes**: 맨 위에 있는 **`repo`** 항목에 반드시 체크합니다. (이게 있어야 저장이 가능합니다.)
7. 맨 아래 **Generate token** 클릭.
8. **⚠️ 주의**: 화면에 나타난 `ghp_...`로 시작하는 긴 코드를 반드시 복사해서 메모장에 저장하세요. **이후에는 다시 볼 수 없습니다.**

> **Tip**: 위 과정이 번거로우시다면 **[pnuai.github.io/pnu-slate/admin/](https://pnuai.github.io/pnu-slate/admin/)** 에 접속하여 이름만 입력하고 토큰을 편리하게 발급받을 수 있습니다!

## 3. 접속 및 탐색: 관리자 페이지(CMS) 인터페이스 익히기

### 3.1 관리자 접속

웹브라우저 주소창에 다음을 입력합니다.

> `https://pnuai.github.io/pnu-slate/admin/`

### 3.2 토큰 등록 (최초 1회)

1. 아까 메모해둔 **GitHub 토큰**을 붙여넣습니다.
2. **저장**을 누릅니다. 이제 브라우저가 여러분을 기억합니다.
3. **주의**: 현재 사용하는 PC와 브라우저에서만 기억됩니다. 그 외의 환경에서는 토큰을 다시 입력하셔야 합니다.

## 4. 실전 편집: 온라인 에디터 활용 가이드

### 4.1 대상 리포지토리 연결

* 관리자 화면에서 작업하고자 하는 대상 레포지토리(브로슈어 이름)를 정확히 입력하고 [변경 저장]을 누르세요.
* 연결이 완료되면 해당 브로슈어의 **파일 목록**이 새로고침됩니다. 이 목록은 브로슈어의 **전체 목차**와 동일합니다.

### 4.2 온라인 편집기 사용 규칙

1. **필수 설정 정보**: 파일을 추가하거나 수정할 때 아래 4가지 항목을 반드시 설정해야 합니다.
* **제목**: 목차에 표시될 공식 명칭입니다.
* **순서**: 문서 간의 정렬 순서이며, 다른 파일과 중복되지 않아야 합니다.
* **ID**: 시스템 구분을 위한 고유값으로, 전체 문서에서 유일(Unique)해야 합니다.
* **색상 테마**: 해당 페이지의 전반적인 분위기를 결정하는 테마를 선택합니다.


2. **⚠️ 마크다운(Markdown) 탭 사용 필수**
* 편집기에는 `Markdown`과 `WYSIWYG` 탭이 있습니다.
* **절대 `WYSIWYG` 탭으로 전환하지 마세요.** 위지윅 모드는 서식이 있는 텍스트를 처리하는 과정에서 코드를 변형시켜 오류를 발생시킬 수 있습니다. 반드시 **Markdown 탭**에서만 수정해 주세요.


3. **실시간 미리보기**: 오른쪽의 미리보기 창을 통해 CSS까지 렌더링된 최종 결과물을 실시간으로 확인하며 작업할 수 있습니다.

### 4.3 콘텐츠 작성 팁

* **파일 확장자**: 오직 `.md` 파일만 지원합니다.
* **HTML/CSS 지원**: 마크다운 문법 외에도 표준 HTML 및 CSS 코드를 직접 입력하여 자유로운 레이아웃을 구성할 수 있습니다.
* **섹션 생성**: 본문 내용 중 `##` (소제목 2)를 기준으로 웹사이트의 섹션이 자동으로 구분됩니다.
* **이미지 첨부**: 현재 버전에서는 관리자 페이지에서 직접 업로드를 지원하지 않습니다.
1. 해당 레포지토리의 `static/img/` 폴더에 이미지가 먼저 추가되어야 합니다.
2. 본문에는 `![설명](/img/파일명.png)` 형식으로 입력합니다.
3. 이미지 추가가 필요할 경우 **개발팀에 요청**하거나 **GitHub 업로드 기능**을 직접 활용하세요.

## 5. 구조 관리: 문서 순서 변경 및 새 페이지 만들기

### 5.1 파일 이름을 통한 순서 제어

모든 문서는 파일 이름 맨 앞에 붙은 **숫자**를 기준으로 순서가 정해집니다.

* **예시**: 현재 `01-intro.md`, `02-part-2.md` 상태에서 `part-2`를 앞으로 보내고 싶다면, 파일 이름을 `00-part-2.md`와 같이 더 낮은 숫자로 변경하여 저장하세요.

### 5.2 페이지 분리와 관리

* 각 파일은 브로슈어의 개별 페이지에 해당합니다.
* 파일 목록에서 **새 문서 작성, 기존 파일 수정, 순서 변경, 파일 업로드** 기능을 통해 구조를 자유롭게 설계할 수 있습니다.
* **주의**: 파일명이 겹치거나 숫자가 중복될 경우 메뉴 탐색 시 혼선이 생길 수 있으므로 규칙적인 번호 부여(예: 10 단위 부여)를 권장합니다.

## 6. 디자인 치트키: 전용 컴포넌트(MDX) 활용 템플릿

### 6.1 하이라이트 (강조 텍스트)

```jsx
<Highlight color="#981B1E">부산대학교 AI융합교육원</Highlight>

```

### 6.2 2단 레이아웃 (반으로 나누기)

```jsx
<Columns>
  <Column>
    ### 교육 대상
    - AI 전공 대학원생
    - 관련 업계 종사자
  </Column>
  <Column>
    ### 교육 장소
    - 부산대학교 장전캠퍼스
  </Column>
</Columns>

```

### 6.3 버튼 및 자료 다운로드

```jsx
<HeroButton href="https://pnuai.github.io">포털로 이동</HeroButton>
<PdfButton href="/pdf/brochure.pdf">브로슈어 다운로드</PdfButton>

```

### 6.4 안내 및 알림 박스 (Callout)

중요한 공지나 주의사항을 시각적으로 강조합니다. `type`에 따라 색상이 바뀝니다.

```jsx
<Callout type="info" title="안내드립니다">
  본 과정은 부산대학교 재학생이라면 누구나 신청 가능합니다.
</Callout>

<Callout type="warning" title="주의">
  신청 기간 이후에는 수정이 불가능합니다.
</Callout>
```

- **지원되는 타입**: `info` (기본), `success` (성공), `warning` (주의), `danger` (위험)

### 6.5 추진 절차 (Step List)

단계별 진행 절차를 깔끔하게 나열할 때 사용합니다.

```jsx
<StepList>
  <StepItem step="1" title="서류 접수">
    온라인 홈페이지를 통해 지원서를 제출합니다.
  </StepItem>
  <StepItem step="2" title="면접 심사">
    1차 합격자를 대상으로 대면 면접을 실시합니다.
  </StepItem>
</StepList>
```

### 6.6 아코디언/FAQ (Details)

많은 내용을 깔끔하게 숨겨두고 클릭 시 보여주거나, FAQ 섹션을 만들 때 사용합니다.

```jsx
<Details summary="Q. 수강료가 있나요?">
  본 교육 과정은 전액 국비 지원으로 무료로 운영됩니다.
</Details>
```

## 7. 최종 배포: 수정 사항을 세상에 공개하기

1. 수정이 마쳤다면 우측 하단의 **[저장 및 배포]** 버튼을 클릭합니다.
2. **기다리기**: GitHub 로봇이 홈페이지를 새로 조립하는 시간 **약 2분**을 기다립니다.
3. 실제 홈페이지([pnuai.github.io](https://pnuai.github.io))에 접속하여 확인합니다.
4. 변경 사항이 즉시 반영되지 않는다면, 브라우저의 강력 새로고침(Ctrl + F5)을 눌러 확인해 주세요.

## 8. 문제 해결: 관리 중 발생하는 상황별 대처법

**Q1. 저장 버튼을 눌렀는데 오류가 나요.**

* **원인**: GitHub 토큰이 잘못되었거나 권한(`repo`) 체크를 안 했을 확률이 높습니다.
* **해결**: [2. 준비 단계](https://www.google.com/search?q=%232-%EC%A4%80%EB%B9%84-%EB%8B%A8%EA%B3%84-%EA%B4%80%EB%A6%AC-%EA%B6%8C%ED%95%9Cgithub-%ED%86%A0%ED%81%B0-%ED%99%95%EB%B3%B4%ED%95%98%EA%B8%B0)를 다시 수행하여 토큰을 재등록하세요.

**Q2. 홈페이지에 반영이 안 돼요.**

* **원인**: 로봇이 아직 빌드 중이거나 브라우저 캐시 문제일 수 있습니다.
* **해결**: 3분 정도 기다린 후 강력 새로고침(Ctrl + Shift + R)을 수행하세요.

**Q3. 화면이 이상하게 깨졌어요.**

* **원인**: `<Columns>` 같은 태그를 열고 닫지 않았을 때 발생합니다.
* **해결**: 모든 열린 태그에 대응하는 닫는 태그(예: `</Columns>`)가 있는지 확인하세요.

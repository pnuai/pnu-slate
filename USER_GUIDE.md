## 1. 기초 이해: PNU Slate의 전체 구조와 작동 원리

PNU Slate는 여러 교육 프로그램을 체계적으로 관리하기 위한 통합 플랫폼으로, 크게 **세 가지 핵심 요소**로 구성되어 있습니다.

### 1.1 서비스 구성 (어떤 페이지들이 있나요?)

*   **PNU AI Hub (허브 페이지)**: 모든 브로슈어 프로젝트가 모여 있는 **메인 입구**입니다. 이곳에서 수정하고자 하는 교육 프로그램(레포지토리)을 선택하여 진입하실 수 있습니다.
*   **Admin Page (관리자 페이지)**: 실제 내용을 수정하고 저장하는 **작업 공간**입니다. 복잡한 프로그래밍 지식이 없으셔도 문서 편집기처럼 쉽고 간편하게 브로슈어를 편집하실 수 있도록 설계되었습니다.
*   **Brochure Page (브로슈어 페이지)**: 편집이 완료되어 일반 사용자에게 공개된 **최종 결과물**입니다. 학생들이 실제 정보를 확인하는 웹사이트입니다.

### 1.2 시스템 작동 원리 (Git 기반 CMS)

PNU Slate는 'Git 기반 CMS'라는 기술을 활용하여 안정성을 확보하고 있습니다. 편집자분께서 [저장] 버튼을 누른 후 실제 사이트에 반영되기까지의 과정은 다음과 같습니다.

1. **저장 및 기록**: 관리자 페이지에서 내용을 수정하고 [저장]을 누르시면, 시스템이 이를 인지하여 소스 코드 저장소(GitHub)에 수정 기록을 안전하게 남깁니다.
2. **자동 제작(Build)**: 새로운 기록이 감지되면, 시스템 내의 자동화 로봇(**GitHub Actions**)이 약 1~2분 동안 홈페이지를 다시 조립합니다.
3. **최종 공개**: 조립이 완료되면 **[pnuai.github.io](https://pnuai.github.io)** 주소를 통해 수정된 내용이 전 세계에 즉시 반영됩니다.

## 2. 준비 단계: 관리 권한(GitHub 토큰) 확보하기

시스템을 안전하게 관리하기 위해서는 "권한이 있는 운영자"임을 증명하는 암호 키(토큰)가 필요합니다. 최초 1회만 설정하시면 됩니다.

### 2.1 개인 액세스 토큰(PAT) 발급 상세 절차

1. [GitHub](https://github.com/) 사이트에 접속하여 로그인해 주십시오.
2. 우측 상단의 **프로필 아이콘**을 클릭한 뒤, **Settings** 메뉴로 들어갑니다.
3. 좌측 메뉴 맨 아래에 있는 **Developer settings**를 클릭해 주십시오.
4. **Personal access tokens** 메뉴 내의 **Tokens (classic)**를 선택합니다.
5. **Generate new token (classic)** 버튼을 클릭합니다.
6. **Note**: 용도를 기억하기 쉽도록 `PNU Slate 관리용`이라고 입력해 주십시오.
7. **Select scopes**: 맨 위에 있는 **`repo`** 항목에 반드시 체크해 주시기 바랍니다. (이 권한이 있어야만 저장이 가능합니다.)
8. 맨 아래의 **Generate token** 버튼을 클릭합니다.
9. **⚠️ 매우 중요**: 화면에 나타난 `ghp_...`로 시작하는 긴 문자열을 반드시 복사하여 별도의 메모장에 저장해 주십시오. **이 창을 닫은 후에는 다시는 확인할 수 없으므로 주의가 필요합니다.**

## 3. 접속 및 탐색: 관리자 페이지(CMS) 인터페이스 익히기

### 3.1 관리자 접속 방법

브로슈어 페이지 상단 네비게이션 바의 **[관리 페이지]** 버튼을 클릭하시거나, 웹브라우저 주소창에 다음 주소를 직접 입력하여 접속해 주시기 바랍니다.

> `https://pnuai.github.io/{레포지토리 이름}/admin/`

### 3.2 토큰 등록 (최초 1회)

1. 발급받으신 **GitHub 토큰**을 입력창에 붙여넣어 주십시오.
2. **저장**을 누르시면 해당 브라우저가 여러분을 관리자로 기억하게 됩니다.
3. **참고**: 현재 사용 중인 PC와 브라우저 환경에만 저장되므로, 다른 PC에서 접속하실 때는 토큰을 다시 한 번 입력해 주셔야 합니다.

## 4. 실전 편집: 온라인 에디터 활용 가이드

### 4.1 대상 리포지토리 연결

*   관리자 화면에서 작업하고자 하는 브로슈어(레포지토리) 이름을 정확히 선택하고 [변경 저장]을 눌러 주십시오.
*   연결이 완료되면 해당 브로슈어의 **파일 목록**이 새로고침됩니다. 이 목록은 실제 브로슈어의 **전체 목차**와 일치합니다.

### 4.2 온라인 편집기 사용 규칙 (중요)

1. **필수 설정 정보**: 새 문서를 만들거나 수정할 때 아래 4가지 항목을 반드시 확인해 주십시오.
    *   **제목**: 목차에 표시될 공식 명칭입니다.
    *   **순서**: 문서 간의 정렬 순서로, 숫자가 낮을수록 앞에 위치합니다.
    *   **ID**: 시스템 구분을 위한 고유 코드로, 다른 문서와 중복되지 않아야 합니다.
    *   **색상 테마**: 페이지의 전반적인 분위기를 결정하는 색상을 선택합니다.

2. **실시간 라이브 스튜디오 (Live Studio) 활용**
    *   이제 에디터는 **좌측(편집기) / 우측(실시간 렌더러)**의 2분할 구조로 동작합니다.
    *   **실시간 동기화**: 좌측에서 마크다운을 입력하면 우측의 실제 브로슈어 화면에 0.1초 내로 즉시 반영됩니다.
    *   **WYSIWYG 경험**: 기존의 단순한 텍스트 미리보기가 아닌, 실제 사이트의 폰트, 레이아웃, 컴포넌트가 그대로 적용된 "최종 결과물"을 보면서 편집할 수 있습니다.
    *   **인쇄 테스트**: 프리뷰 상단의 **[🖨 인쇄 테스트]** 버튼을 누르면 A4 용지 규격에 맞춰진 인쇄 미리보기가 실행되어 PDF 저장 시의 결과물을 미리 확인할 수 있습니다.

3. **마크다운(Markdown) 집중 환경**
    *   새로운 스튜디오는 마크다운 편집기에 최적화되어 있습니다. 기존의 '위지윅(WYSIWYG)' 탭은 디자인 코드의 오동작을 방지하기 위해 제거되었으며, 대신 우측의 강력한 라이브 렌더러가 그 역할을 완벽히 대체합니다.

### 4.3 콘텐츠 작성 팁

*   **파일 확장자**: 시스템은 오직 `.md` 파일만을 문서로 인식합니다.
*   **섹션 자동 생성**: 본문 중 `##` (소제목)을 입력하시면, 우측 목차(TOC)에 섹션이 자동으로 구분되어 표시됩니다.
*   **이미지 활용**: 현재 버전은 관리자 페이지에서의 직접 업로드를 지원하지 않습니다. 
    1. 이미지를 해당 레포지토리의 `static/images/` 폴더에 먼저 업로드해야 합니다.
    2. 본문에는 `![이미지 설명](/images/파일명.png)` 형식으로 입력해 주십시오.
    3. 이미지 추가가 필요할 경우 **기술 지원팀에 요청**하시거나 **GitHub 업로드 기능**을 활용해 주시기 바랍니다.

## 5. 구조 관리: 문서 순서 변경 및 새 페이지 만들기

### 5.1 파일 이름을 통한 순서 제어

모든 문서는 파일 이름 맨 앞에 붙은 **숫자**를 기준으로 순서가 정해집니다.

*   **예시**: 현재 `01-개요.md`, `02-교육내용.md` 상태에서 '교육내용'을 앞으로 보내고 싶다면, 파일 이름을 `00-교육내용.md`와 같이 더 낮은 숫자로 변경하여 저장해 주십시오.

### 5.2 페이지 관리 팁

*   새로운 페이지를 추가하실 때는 [새 문서 작성] 버튼을 활용해 주십시오.
*   문서들 사이에 나중에 새로운 내용을 끼워 넣기 편리하도록, 번호를 **10단위(예: 10, 20, 30...)**로 부여하시는 것을 권장해 드립니다.

## 6. 디자인 치트키: 전용 컴포넌트 활용하기

브로슈어의 완성도를 높여주는 세련된 디자인 도구들이 준비되어 있습니다. 아래 코드를 복사하여 필요한 부분의 텍스트만 수정해 보세요.

### 6.1 하이라이트 (강조 텍스트)
문장 속에서 특정 단어를 강조할 때 유용합니다.
```jsx
<Highlight color="#1a4fa0">강조할 텍스트</Highlight>
```

### 6.2 브로슈어 전용 컬럼 (Stable Columns)
인쇄 및 PDF 출력 시에도 줄바꿈이 깨지지 않도록 설계된 전용 컬럼 시스템입니다. 2단 또는 3단 구성을 선택할 수 있습니다.
```jsx
<div className="brochure-columns-2">
  <div className="brochure-col">
    ### 왼쪽 영역
    내용을 입력해 주세요.
  </div>
  <div className="brochure-col">
    ### 오른쪽 영역
    내용을 입력해 주세요.
  </div>
</div>
```
*   **3단 구성 시**: `brochure-columns-3` 클래스를 사용하고 내부에 `brochure-col` 세 개를 넣습니다.

### 6.3 버튼 및 자료 연결
```jsx
<HeroButton href="링크주소">홈페이지 바로가기</HeroButton>
```

### 6.4 안내 및 알림 박스 (Callout)
중요한 공지나 주의사항을 시각적으로 강조합니다.
```jsx
<Callout type="info" title="안내">본 과정은 누구나 신청 가능합니다.</Callout>
```
*   **타입**: `info`(파랑), `success`(초록), `warning`(노랑), `danger`(빨강)

### 6.5 추진 절차 (Step List)
단계별로 진행되는 절차를 예쁘게 나열할 때 사용합니다.
```jsx
<StepList>
  <StepItem step="1" title="신청 접수">내용을 입력하세요.</StepItem>
  <StepItem step="2" title="결과 발표">내용을 입력하세요.</StepItem>
</StepList>
```

### 6.6 아코디언/FAQ (Details)
많은 내용을 숨겨두었다가 클릭 시 보여주고 싶을 때 사용합니다.
```jsx
<Details summary="Q. 수강료는 얼마인가요?">본 교육은 전액 국비 지원으로 무료입니다.</Details>
```

### 6.7 글자 크기와 폰트 조절 (Text)
다양한 서체와 크기를 자유롭게 조절하고 싶을 때 사용합니다. 숫자만 입력하면 크기가 조절되어 매우 편리합니다.
```jsx
<Text size="32" weight="700" family="gmarket" color="#1a4fa0">큼직한 제목</Text>
```
*   **size**: 원하는 픽셀 숫자만 입력 (예: 24, 40, 60 등)
*   **weight**: 글자 굵기 (100~900 사이, 숫자가 클수록 두꺼움)
*   **family**: `gmarket`(지마켓 산스), `pretendard`(프리텐다드), `nanum`(나눔고딕), `noto`(노토산스)

## 7. 최종 배포 및 뷰어 설정

### 7.1 수정 사항 실제 반영하기
1. 편집을 모두 마치셨다면 우측 하단의 **[저장 및 배포]** 버튼을 클릭해 주십시오.
2. 시스템이 새로운 홈페이지를 제작하는 데 **약 1~2분** 정도의 시간이 소요됩니다.
3. 잠시 후 실제 홈페이지에 접속하여 반영된 내용을 확인해 주시기 바랍니다.
4. 내용이 보이지 않는다면 브라우저에서 `Ctrl + F5`를 눌러 새로고침해 주십시오.

### 7.2 레이아웃 규격 선택 기능
브로슈어 우측 하단의 **종이 모양 아이콘**을 클릭하면, 현재 화면의 가로 너비를 실제 판촉물 규격에 맞춰 변경할 수 있습니다.
*   **A4 브로슈어**: 가장 표준적인 문서 느낌의 규격입니다.
*   **A3 포스터**: 정보를 시원하고 넓게 보여줄 때 적합합니다.
*   **B5 리플렛**: 컴팩트하고 아기자기한 느낌을 줍니다.
*   **규격 저장**: 한 번 설정한 규격은 해당 기기에 저장되어, 다음번에 접속하실 때도 동일하게 유지됩니다.

## 8. 문제 해결 안내 (Q&A)

**Q1. 저장 버튼을 눌렀는데 '권한 오류'가 발생합니다.**
*   GitHub 토큰이 올바르지 않거나, 발급 시 `repo` 권한을 체크하지 않았을 가능성이 큽니다. [2. 준비 단계]를 참고하여 토큰을 다시 등록해 주시기 바랍니다.

**Q2. 홈페이지에 수정 내용이 바로 나타나지 않습니다.**
*   시스템 자동 조립에 1~2분 정도의 시간이 걸립니다. 잠시만 기다려 주신 후 강력 새로고침(`Ctrl + Shift + R`)을 시도해 보세요.

**Q3. 화면 레이아웃이 갑자기 깨졌습니다.**
*   컴포넌트 태그를 열고 제대로 닫지 않았을 때 주로 발생합니다. `<Columns>`로 시작했다면 반드시 마지막에 `</Columns>`가 있는지 확인해 주십시오.

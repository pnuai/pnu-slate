# 📘 PNU Slate 가이드북 (Guidebook)

안녕하세요! **PNU Slate**를 통해 부산대학교 AI융합교육원의 온라인 브로슈어를 관리하게 된 것을 환영합니다. 🎉  
이 문서는 코딩을 모르는 분들도 마우스 클릭 몇 번으로 멋진 브로슈어를 만들고 수정할 수 있도록 돕는 **단계별 친절 가이드**입니다.

---

## 🧭 시작하기 전에: 서비스 작동 원리

수정 버튼을 누르면 어떤 일이 일어나는지 궁금하신가요?
1. **관리자 페이지**에서 내용을 고치고 **[저장 및 배포]**를 누릅니다.
2. 시스템이 자동으로 여러분의 수정 사항을 **GitHub 저장소**로 보냅니다.
3. GitHub의 로봇(Actions)이 새로운 코드를 읽고 **1~2분 동안 웹사이트를 다시 빌드**합니다.
4. 빌드가 끝나면 전 세계 누구나 여러분이 수정한 내용을 볼 수 있게 됩니다!

---

## 1단계: 관리자 에디터 접속하기 💻

브로슈어 내용을 수정하고 싶을 때는 언제 어디서든 웹브라우저만 있으면 됩니다.

1. **바로 접속**: 홈페이지 주소 끝에 `/admin`을 입력하세요.
   - 예: `https://pnuai.github.io/pnu-slate/admin/`
2. **문서에서 접속**: 브로슈어를 읽다가 맨 아래로 내려가면 **[이 페이지 편집하기]** 버튼이 있습니다. 이 버튼을 누르면 지금 보고 있는 페이지가 에디터에 자동으로 열립니다.

---

## 2단계: 필수 설정 - GitHub 토큰(PAT) 등록 ⚙️

문서를 저장하기 위해서는 여러분이 "관리자"라는 증거가 필요합니다. 딱 한 번만 설정하면 브라우저가 기억합니다.

1. **설정창 열기**: 에디터 우측 상단의 **톱니바퀴 아이콘**을 클릭하세요.
2. **토큰 발급**: `[토큰 발급받으러 가기]` 링크를 클릭합니다. (GitHub 로그인이 필요합니다)
3. **권한 체크**: 토큰 생성 화면에서 **`repo`** 항목에 반드시 체크하세요!
4. **복사 및 저장**: 생성된 `ghp_...` 로 시작하는 긴 문자열을 복사해서 설정창의 빈칸에 붙여넣고 **[저장]**을 누르세요.

---

## 3단계: 내용 수정 및 실시간 미리보기 🎨

**PNU Slate**의 에디터는 수정하는 즉시 실제 디자인을 보여줍니다.

- **Markdown 탭 사용**: 반드시 **`Markdown`** 탭에서 작업하세요! (WYSIWYG 탭은 복잡한 디자인을 깨뜨릴 수 있습니다)
- **글자만 바꾸기**: 복잡해 보이는 코드 사이에서 **실제 내용(한글)**만 찾아서 고치면 됩니다.
- **실시간 프리뷰**: 오른쪽 화면에서 디자인이 실시간으로 변하는 것을 확인하며 작업하세요. 실제 웹사이트와 100% 동일하게 보입니다.

---

## 4단계: 저장 및 배포 🚀 (가장 중요!)

수정이 끝났다면 세상에 공개할 차례입니다.

1. 우측 하단의 **[저장 및 배포]** 버튼을 누르세요.
2. **"문서 저장 및 배포가 시작되었습니다."** 메시지가 나오면 성공입니다.
3. **기다림의 미학**: GitHub 로봇이 웹사이트를 새로 만드는 데 **최소 1분~2분**이 걸립니다. 
   - *팁: 작업이 queued(대기 중) 상태라면 다른 사용자의 작업이 처리 중일 수 있습니다. 조금만 기다려 주세요!*
4. 잠시 후 홈페이지를 새로고침(F5) 하면 여러분의 작품이 반영되어 있을 것입니다.

## ✨ 브로슈어 디자인 템플릿 (복사해서 쓰세요!)

가이드북에 활기를 불어넣고 싶다면 아래 코드를 복사해서 **Markdown** 탭에 붙여넣으세요.

### 1. 예쁜 하이라이트 글씨
```jsx
<Highlight color="#1651c8">파란색 강조</Highlight>
<Highlight color="#e84b35">빨간색 강조</Highlight>
```

### 2. 반응형 2단 레이아웃 (반으로 나누기)
```jsx
<Columns>
  <Column>
    <h3>왼쪽 제목</h3>
    <p>내용을 입력하세요.</p>
  </Column>
  <Column>
    <h3>오른쪽 제목</h3>
    <p>내용을 입력하세요.</p>
  </Column>
</Columns>
```

## 🆘 도움이 필요하신가요? (FAQ)

**Q. 배포가 너무 늦어요!**  
A. GitHub의 서버 상황에 따라 가끔 늦어질 수 있습니다. 보통 2~3분 내에는 완료되니 잠시만 기다려 주세요.

**Q. 실수로 이상한 코드를 지웠어요!**  
A. 당황하지 마세요! 저장 버튼을 누르기 전이라면 페이지를 새로고침하면 이전 상태로 돌아갑니다.

**Q. 이미지가 안 나와요.**  
A. 이미지는 `static/img` 폴더에 업로드되어 있어야 합니다. 개발자에게 문의하거나 가이드의 이미지 경로를 확인해 보세요.
